<Link>:

역할: 클릭 시 경로 이동을 돕는 네비게이션 컴포넌트.
사용 위치: 네비게이션 메뉴, 버튼, 이미지 등.
기능: URL 변경 후 해당 경로의 컴포넌트 렌더링.

<Route>:

역할: 경로와 컴포넌트 매핑.
사용 위치: 라우팅 설정 시, 주로 최상위 컴포넌트에서 사용.
기능: URL 경로에 맞는 컴포넌트 렌더링.

**더 큰 차이점**
< UI 구성에 대한 역할 차이 >

<Link>는 화면에 표시되어 사용자가 클릭할 수 있는 인터페이스 요소를 제공합니다. 
말 그대로 클릭 가능한 '링크'죠.

<Route>는 실제로 특정 경로에 도달했을 때 어떤 콘텐츠를 보여줄지를 결정합니다.
즉, 실제 화면에 렌더링될 콘텐츠와 직접적으로 관련됩니다.

< 상호작용의 시작과 끝 >

<Link>는 사용자가 페이지를 탐색하는 과정에서 상호작용의 시작을 담당합니다. 예를 들어, 사용자가 어떤 메뉴 항목을 클릭해서 다른 페이지로 이동하려는 순간입니다.
<Route>는 사용자가 페이지 탐색을 통해 도달한 결과입니다. 사용자가 특정 링크를 통해 이동한 후에 보여줄 최종 화면을 정의합니다.

**추가 비유**

<Link>는 엘리베이터 버튼: 특정 층으로 가기 위해 누르는 버튼과 같습니다.
<Route>는 해당 층의 로비: 엘리베이터가 멈춘 후 열리는 층의 로비입니다.

<Link>: 사용자 인터페이스의 링크로 사용되어 사용자가 클릭했을 때 경로를 변경합니다. (사용자 인터페이스 측면)
<Route>: URL 경로에 따라 어떤 컴포넌트를 렌더링할지를 설정합니다. (라우팅 설정 측면)
useNavigate(): 코드에서 프로그래밍적으로 경로를 변경합니다. (프로그래밍적 경로 변경)
