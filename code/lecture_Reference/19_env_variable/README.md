# `dotenv`와 `cross-env`
Node.js 환경에서 환경 변수를 관리하는데 사용되는 라이브러리입니다.


## `dotenv`
- **`.env` 파일을 통해 환경 변수를 로드**하고 Node.js 어플리케이션에 사용할 수 있게 해줍니다.
- 이는 개발, 테스트, 프로덕션 등 다양한 환경에서 서로 다른 설정을 사용하도록 도와줍니다.
- 장점
  - 쉽게 설정 파일을 만들어서 프로젝트에 적용할 수 있습니다.
  - 민감한 정보(비밀번호, API 키 등)를 코드에서 분리하여 보안을 유지하는데 도움이 됩니다.
  - 버전 관리 시스템(Git 등)에 민감한 정보가 포함되지 않도록 합니다.
- 단점
  - `.env` 파일 자체가 외부로 유출되면 보안 문제가 발생할 수 있으므로 주의해야 합니다.
  - dotenv는 기본적으로 비동기 API를 제공하지 않습니다. 이 때문에 비동기 작업이 필요한 경우 별도의 처리가 필요합니다.

## `cross-env`
- Node.js 스크립트를 실행할 때 **OS 별 차이 없이 일관성있게 환경 변수를 설정**할 수 있게 해주는 패키지입니다.
- 장점
  - Windows, Linux, macOS 등 다양한 운영 체제에서 스크립트 실행 시 동일하게 환경 변수를 설정해줍니다. 이렇게 함으로써 OS간 호환성 문제 없이 애플리케이션을 구동시킬 수 있습니다.
  - 개발자들 사이에서 코드나 명령어의 일관성을 유지하는데 도움됩니다.
- 단점
  - cross-env 자체에 큰 단점은 없지만, 실제 운영환경에서는 대개 환경변수가 이미 설정되어 있거나 다른 방식으로 관리되므로 굳이 필요하지 않을 수 있습니다.


두 패키지 모두 환경 변수를 관리하는 도구이지만, 사용 목적과 방식에 차이가 있습니다. 

## TL;DR
- `dotenv`: 환경변수를 파일로부터 로드하고 관리
- `cross-env`: 운영체제 간 차이 없이 일관성있게 환경변수를 설정하는데 초점

## package.json
```json
"scripts": {
  "start": "cross-env NODE_ENV=development node index",
  "start:prod": "cross-env NODE_ENV=production node index",
  "test": "echo \"Error: no test specified\" && exit 1"
}
```

- scripts 섹션: 여러 npm 스크립트를 정의하는 곳
- `npm run <스크립트명>` 명령으로 실행 가능

### "start": "cross-env NODE_ENV=production PORT=80 node index" 부분을 해석

- `start`: "start"는 기본적으로 Node.js 애플리케이션을 시작하는 데 사용되는 스크립트 이름입니다. 이 이름은 Node.js 커뮤니티에서 널리 인식되고 있으며, 특별한 설정 없이 npm start 명령으로 실행할 수 있습니다.
- `cross-env`: 이 부분은 cross-env 패키지를 사용하여 환경 변수를 설정하는 것을 의미합니다. cross-env 패키지는 다양한 운영체제에서 일관된 방식으로 환경 변수를 설정하게 해주는 도구입니다.
- `NODE_ENV=production PORT=80`: cross-env 뒤에 나오는 부분이 실제로 설정하려는 환경 변수들입니다. 여기서는 NODE_ENV를 'production'으로, PORT를 '80'으로 설정하라고 지시하고 있습니다.
- `node index`: 마지막 부분인 'node index'는 실제로 실행될 Node.js 코드 파일을 가리킵니다. 이 경우, 프로젝트의 루트 디렉터리에 'index.js'라는 이름의 파일이 존재해야 하며, 해당 파일이 애플리케이션 서버의 주요 로직을 포함하게 됩니다.
따라서 전체적으로 보면, "start" 스크립트는 서버가 프로덕션 모드로 작동하도록 하고(port 80), 그러한 상태에서 'index.js'라는 애플리케이션 코드를 실행하는 것을 의미합니다.
