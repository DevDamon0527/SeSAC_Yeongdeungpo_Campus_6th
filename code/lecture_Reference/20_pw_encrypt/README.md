# 암호화

## crypto 모듈

- Node.js의 내장 모듈입니다.
- 암호화 기능을 제공합니다.
- 해시 함수, HMAC, 암호화, 복호화, 서명 등 다양한 암호화 작업을 수행할 수 있습니다.
- 주로 비밀번호 해싱, 데이터 무결성 검증 등에 사용됩니다.

## bcrypt 모듈

- 외부 라이브러리로, 설치가 필요합니다.
- 비밀번호 해싱에 특화된 모듈입니다.
- 솔트(salt)를 자동으로 생성하고 적용합니다.
- 느린 해싱 알고리즘을 사용하여 무차별 대입 공격에 강합니다.

> 두 모듈의 주요 차이점은 용도와 보안성입니다. crypto는 범용적인 암호화 기능을 제공하는 반면, bcrypt는 비밀번호 저장에 특화되어 있고 더 높은 보안성을 제공합니다.

## crypto vs. bcrypt

- crypto 모듈

  - 양방향 암호화: 지원합니다.
  - 예: AES, DES 등의 알고리즘을 사용한 암호화/복호화
  - 단방향 암호화(해싱): 지원합니다.
  - 예: MD5, SHA256 등의 해시 함수

- bcrypt 모듈

  - 양방향 암호화: 지원하지 않습니다.
  - 단방향 암호화(해싱): 지원합니다.
  - 이는 bcrypt의 주요 기능입니다.

> crypto 모듈은 다양한 암호화 기능을 제공하므로 양방향과 단방향 암호화를 모두 지원합니다. 반면 bcrypt는 비밀번호 해싱에 특화된 모듈로, 오직 단방향 암호화만을 제공합니다.
> bcrypt가 단방향 암호화만 제공하는 이유는 비밀번호와 같은 민감한 정보는 복호화할 필요가 없고, 오히려 복호화가 가능하면 보안상 위험하기 때문입니다.

## 참고. Buffer

Buffer는 Node.js에서 이진 데이터를 다루기 위한 객체입니다.

raw 바이너리 데이터를 표현하는 Node.js의 방식으로, 다양한 형태의 데이터를 효율적으로 다룰 수 있게 해줍니다.

### 특징

- 고정된 크기의 메모리 영역을 나타냅니다.
- 주로 파일 시스템 작업, 네트워크 통신, 암호화 등에서 사용됩니다.

### 데이터 표

- Buffer는 16진수 형태의 바이트 시퀀스로 데이터를 저장합니다.
- 예: <Buffer 68 65 6c 6c 6f>는 "hello"를 나타냅니다.

### 사용 이유

- 이진 데이터를 효율적으로 처리할 수 있습니다.
- 문자열로 직접 표현하기 어려운 데이터를 다룰 수 있습니다.

### 변환

- toString() 메서드를 사용하여 다양한 인코딩(예: 'base64', 'hex', 'utf8')으로 변환할 수 있습니다.
