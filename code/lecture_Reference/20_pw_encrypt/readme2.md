# 비밀번호 암호화와 보안 기술


## 1. 단방향 암호화 (One-way Encryption)
단방향 암호화는 원본 데이터를 암호화된 형태로 변환하지만, 암호화된 데이터를 다시 원본으로 되돌릴 수 없는 방식입니다.

### 예제

- 원본: "password123"
- 암호화 결과: "5f4dcc3b5aa765d61d8327deb882cf99"

### 특징

- 복호화가 불가능합니다.
- 비밀번호 저장에 주로 사용됩니다.
- 로그인 시 사용자가 입력한 비밀번호를 암호화하여 저장된 값과 비교합니다.

### 비유 
단방향 암호화는 달걀을 깨서 오믈렛을 만드는 것과 비슷합니다. 오믈렛을 다시 달걀로 되돌릴 수 없듯이, 암호화된 결과를 원래 비밀번호로 되돌릴 수 없습니다.


## 2. 양방향 암호화 (Two-way Encryption)
양방향 암호화는 원본 데이터를 암호화된 형태로 변환하고, 필요할 때 다시 원본 데이터로 복호화할 수 있는 방식입니다.

### 예제

- 원본: "confidential_data"
- 암호화 결과: "U2FsdGVkX1+9v1EcMzZ6tXqUfTwltVCNbg=="
- 복호화 결과: "confidential_data"

### 특징

- 암호화와 복호화가 모두 가능합니다.
- 중요한 정보를 안전하게 저장하고 필요할 때 읽을 수 있어야 할 때 사용합니다.
- 암호화 키가 필요하며, 이 키의 관리가 중요합니다.

### 비유
양방향 암호화는 금고에 물건을 넣고 잠그는 것과 비슷합니다. 금고에 물건을 넣고 잠그면(암호화) 안전하게 보관할 수 있고, 필요할 때 열쇠로 금고를 열어(복호화) 물건을 다시 꺼낼 수 있습니다.


## 3. 해싱 (Hashing)
해싱은 임의의 길이의 데이터를 고정된 길이의 데이터로 변환하는 과정입니다.

### 예제

- 입력: "password123"
- 해시 결과: "ef92b778bafe771e89245b89ecbc08a44a4e166c06659911881f383d4473e94f"

### 특징

- 단방향 과정으로, 원본 데이터로 되돌릴 수 없습니다.
- 같은 입력에 대해 항상 같은 결과를 출력합니다.
- 작은 변화에도 완전히 다른 결과가 나옵니다.

### 비유
해싱은 과일 주스를 만드는 것과 비슷합니다. 여러 과일(데이터)을 넣고 갈면 주스(해시)가 나오지만, 주스를 다시 원래의 과일로 만들 수는 없습니다.

### 사용 예:

- 비밀번호 저장
- 데이터 무결성 검증

## 4. 대칭키 암호화 (Symmetric Key Encryption)
대칭키 암호화는 하나의 키로 암호화와 복호화를 모두 수행하는 방식입니다.

### 예제

- 원본: "Hello, World!"
- 암호화 키: "secretkey"
- 암호화 결과: "2F6C1BD44F50F3D7C630"
- 복호화 결과: "Hello, World!"

### 특징

- 암호화와 복호화에 같은 키를 사용합니다.
- 빠른 처리 속도를 가집니다.
- 키 공유가 안전하게 이루어져야 합니다.

### 비유
대칭키 암호화는 자물쇠와 열쇠 한 쌍을 사용하는 것과 같습니다. 같은 열쇠로 잠그고 열 수 있지만, 열쇠를 안전하게 공유해야 합니다.

### 사용 예

- 파일 암호화
- 데이터베이스 필드 암호화

## 5. 공개키 암호화 (Public Key Encryption)
공개키 암호화는 두 개의 키(공개키와 개인키)를 사용하는 비대칭 암호화 방식입니다.
### 예제

- 원본: "Confidential Message"
- 공개키로 암호화: "A7B3C1D9E5F2G8H6I4J0"
- 개인키로 복호화: "Confidential Message"

### 특징

- 공개키로 암호화하고 개인키로 복호화합니다.
- 키 교환 문제를 해결합니다.
- 대칭키 암호화보다 처리 속도가 느립니다.

### 비유
- 공개키 암호화는 우체통과 같습니다. 누구나 편지(데이터)를 넣을 수 있지만(공개키로 암호화), 열쇠를 가진 사람만 우체통을 열어 편지를 읽을 수 있습니다(개인키로 복호화).

### 사용 예:

- 안전한 통신 (HTTPS)
- 디지털 서명

# 실제 적용 (Node.js)

- `해싱`: bcrypt를 사용한 비밀번호 해싱
- `대칭키 암호화`: crypto 모듈의 AES 암호화 사용
- `공개키 암호화`: crypto 모듈의 RSA 암호화 사용

이러한 개념들을 이해하면 다양한 암호화 기술의 특징과 용도를 파악하고, 적절한 상황에 맞는 기술을 선택할 수 있습니다.
